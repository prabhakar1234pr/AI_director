# AI Director

Turn natural-language scene descriptions into a full mini-film: a screenplay,
a comic-book storyboard, narrated slideshows, and AI-generated video clips —
all driven by a Cursor-style chat that takes direct actions on your project.

## What you get

- **Script** — Celtx-style screenplay generated by Gemini, editable as JSON.
- **Visuals** — one cinematic frame per shot via Gemini image generation, with
  per-shot regenerate and "add to chat → describe a tweak" flow.
- **Storyboard** — a real comic-book page with multiple layout variants,
  AI-picked text-box placement (saliency-based) and drag-to-reposition.
- **Narration** — Google Cloud TTS Chirp 3 HD voices, cast per shot by Gemini
  (narrator + per-character voices) and played as a synced slideshow.
- **AI Video** — Veo 3 Fast image-to-video clips (8s each) auto-played in
  sequence with the Chirp 3 HD narration layered on top.
- **Director Chat** — a single chat that knows which page you're on and edits
  shots, regenerates images, reorders, deletes, etc. directly.

## Stack

- **Frontend**: Next.js 16 (App Router) · React 18 · Tailwind · Zustand
  (persisted to IndexedDB so refreshes keep your work).
- **Backend**: FastAPI (Python 3.12).
- **Script + Chat + Voice casting**: Vertex AI Gemini (`gemini-2.5-flash`).
- **Images**: Vertex AI Gemini (`gemini-2.5-flash-image`).
- **Audio**: Google Cloud Text-to-Speech, Chirp 3 HD voices.
- **Video**: Vertex AI Veo 3 Fast (`veo-3.0-fast-generate-001`).
- **Auth**: a single GCP service account (Workload Identity Federation in
  prod, key file locally) — no end-user auth, no database, no cloud storage.
  Generated media is shipped back to the browser as base64 and persisted in
  IndexedDB.

## Quick start

### 1. Environment

```bash
cp .env.example backend/.env
# Edit backend/.env — set GCP_PROJECT_ID, GCP_LOCATION,
# and GOOGLE_APPLICATION_CREDENTIALS (path to a service-account JSON
# with Vertex AI + Cloud TTS access).
```

### 2. Backend

```bash
cd backend
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

### 3. Frontend

```bash
cd frontend
npm install
npm run dev
# Open http://localhost:3000
```

## Usage

1. Open the chat in the bottom-right and pitch a scene
   (e.g. *"A detective walks through rain-soaked neon streets at midnight"*).
2. Answer at most one or two clarifying questions (style, mood).
3. Review / edit the screenplay on the **Script** tab, or click *Edit JSON*.
4. Switch to **Visuals** → *Generate Visuals* for one image per shot. Hover a
   shot and *Add to chat* to describe a tweak (e.g. *"warmer lighting, more
   fog"*) — the chat regenerates just that frame.
5. **Storyboard** lays the same images out as a comic page; pick a layout
   variant and drag any caption / speech bubble to reposition.
6. **Narration** → *Generate Audio* casts a Chirp 3 HD voice per shot and
   plays a synced slideshow.
7. **AI Video** → *Generate Videos* renders each shot with Veo 3 Fast (1–2 min
   per clip) and auto-plays them with the narration on top.

The director chat works on every page: ask it to *swap shots 1 and 2*, *make
shot 3 darker*, *rewrite the last narration in past tense*, etc. — it edits
your project in place.

## Environment variables

| Variable | Required | Description |
|---|---|---|
| `GCP_PROJECT_ID` | yes | GCP project hosting Vertex AI (Gemini, Veo) |
| `GCP_LOCATION` | yes | Vertex AI region (e.g. `us-central1`) |
| `GOOGLE_APPLICATION_CREDENTIALS` | local only | Path to a service-account JSON. Cloud Run uses the attached service account instead. |
| `FRONTEND_URL` | prod only | Browser origin allowed by CORS (e.g. your Vercel URL) |
| `GEMINI_TEXT_MODEL` | optional | Override script/chat/casting model (default `gemini-2.5-flash`) |
| `GEMINI_IMAGE_MODEL` | optional | Override image model (default `gemini-2.5-flash-image`) |
| `VEO_MODEL` | optional | Override video model (default `veo-3.0-fast-generate-001`) |

The frontend reads `NEXT_PUBLIC_API_BASE` (full `/api` URL) or
`NEXT_PUBLIC_API_URL` (origin) at build time; both default to
`http://localhost:8000/api`.

## Deployment

The backend ships as a Docker image to Google Cloud Run via
`.github/workflows/backend-deploy.yml`, authenticating with Workload Identity
Federation. The frontend is a vanilla Next.js app and deploys cleanly to
Vercel.

## Repo layout

```

  main.py                 # FastAPI app + CORS
  routers/                # script · images · audio · video · chat
  services/
    gemini_service.py     # Vertex Gemini: script, image, chat, voice casting

  models/schemas.py       # Pydantic request / response types
  Dockerfile              # Cloud Run image (uv-based)

  page.jsx                # single-screen playground
  stores/useDirectorStore.js   # Zustand store, persisted to IndexedDB
  components/             # ChatPanel, ScriptPanel, CeltxView,
                          # ComicStoryboard, ShotCard, SlideshowPlayer,
                          # VideoSlideshow, widgets/*
.github/workflows/
  backend-deploy.yml      # Cloud Run deploy via Workload Identity Federation
```
